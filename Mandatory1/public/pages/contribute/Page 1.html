What is Node.js?

An asynchronous event-driven JavaScript runtime. Node.js is designed to build scalable network applications.

Run your app.js through the terminal if you type:

$ node app.js

Or with nodemon installed:

$ nodemon app.js

Nodemon is a tool that helps develop Node.js based applications by automatically restarting the node application when file changes in the directory are detected.

Install with npm:

$ npm install -g nodemon

Installed as development dependency:

$ npm install --save-dev nodemon

Scoping

Scope determines the accessibility (visibility) of variables. JavaScript has 4 types of scope:

- Block scope: Before ES6 (2015), JavaScript had only Global Scope and Function Scope. ES6 introduced two important new JavaScript keywords: let and const. These two keywords provide Block Scope in JavaScript. Variables declared inside a { } block cannot be accessed from outside the block:

- Function scope: Local variables have Function Scope: They can only be accessed from within the function.

- Global scope: A global variable has Global Scope: All scripts and functions on a web page can access it.

Functions and hoisting

Variables defined with let and const are hoisted to the top of the block, but not initialized. Meaning: The block of code is aware of the variable, but it cannot be used until it has been declared. Using a let variable before it is declared will result in a ReferenceError. The variable is in a "temporal dead zone" from the start of the block until it is declared: Declare Your Variables At the Top ! Hoisting is (to many developers) an unknown or overlooked behavior of JavaScript. If a developer doesn't understand hoisting, programs may contain bugs (errors). To avoid bugs, always declare all variables at the beginning of every scope. Since this is how JavaScript interprets the code, it is always a good rule.

Clean code

Writing clean code improves the maintainability of the application and make developers productive. Clean coding means that in the first place you write code for your later self and for your co-workers and not for the machine. Use intention-revealing names when naming variables. Make your variable names easy to pronounce, Functions should do one thing. They should do it well. They should do it only. Use long, descriptive names when you write functions.
